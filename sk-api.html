https://manage.storykeeperevents.com/api/v1

Storykeeper Events API

This API gives programmatic access to manipulate data and configuration for the Storykeeper Management Software

This API is designed to be consumed both internally, and externally. The primary external client will be an AngularJS application.

API Authentication ¶

All requests to this API must include a valid, signed JWT in the authorization header. The JWT claims must bear a signature issued by the authentication service.

Requests issued without a signed and authorized token will receive a 403 response.

Common Data Structure ¶

All responses are wrapped in a common data structure

{
    "data": [
        "Response DATA"
    ],
    "meta": {},
    "pagination": {}
}

meta ¶

Meta data will always be a single level key value structure

{
  "meta": {
    "key1": "value1"
  }
}

Pagination ¶

All List screens will be formatted with pagination

{
    "paginate": {
        "total": 3,
        "per_page": 15,
        "current_page": 1,
        "last_page": 1,
        "next_page_url": null,
        "prev_page_url": null,
        "from": 1,
        "to": 3
    },
}
